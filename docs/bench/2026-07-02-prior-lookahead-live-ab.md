# 世界模型先验 × lookahead：live A/B 量化报告

> 2026-07-02。模型 deepseek-v4-pro。台子 `examples/live-bench.ts`。
> 回答开放线 #1 的世界模型/lookahead 半边："先验到底省不省"，此前只有孤例（5→4 回合，N=1）。

## 方法

- **对照**：同一任务跑两遍——**冷**（全新 WorldModel，此跑顺带训练它）vs **暖**（带上一跑学到的先验）。每组（rep）用全新 WorldModel，页面在两跑间重置；reps=3。
- **任务**：T1 三连动作（3 个独立"完成"键，diff=两个 surface 变化）；T2 表单（两控件 setControl + 提交 invoke）。都是低危、可验证、多步。
- **计量**：LLM 往返数（adapter.step）、HTTP 请求数（含重试，用于排噪）、prompt/completion token（截 API usage）、predict 尝试/命中、批量度（工具调用数/回合）、outcome。
- **过程诚实**：一共试了 4 组配置才定线，中途的失败数据一并保留（见下）。

## 配置探索（每格为 T1 冷/暖 的 rounds 均值 | token 均值）

| 配置 | 冷 | 暖 | 发现 |
|---|---|---|---|
| A 无任何批量鼓励 | 3.67 \| 6020 | 5.00 \| 9205 | 先验**负收益**：lookahead 通路闲置（模型默认一回合一调用），先验只是加长上下文 |
| B 系统提示全局鼓励批量+predict | 6.67 \| 13835 | **3.33 \| 6338** | 暖跑大赚；**但冷跑瞎猜 predict 全落空（0/2）**，批次反复中断，比 A 冷还贵 78% |
| C 改为"有依据才 predict" | 5.33 \| 10464 | 4.00 \| 7725 | 冷跑仍手痒（0/1.33 全落空），措辞挡不住 |
| **D 鼓励语只随先验注入（定稿）** | **4.00 \| 6750** | **3.67 \| 7004** | 冷跑回归基线且 **0 次瞎猜**；暖跑火力保留 |

**教训（写进内核的那条）**：无知识的投机是负收益，有知识的投机是纯赚——鼓励批量的时机必须与知识同源。落地即：批量+predict 的提示不进系统提示，只作为世界模型先验注入块的一部分出现。

| 指标 | T1 冷 | T1 暖 | Δ | T2 冷 | T2 暖 | Δ |

| token（p+c） | 6750 | 7004 | +4% | 8522 | 6544 | **-23%** |
| predict 命中 | 0/0 | 3/3 | — | 0/0 | 1.67/1.67 | — |
| 批量度 | 1.33 | 1.36 | | 1.00 | 1.22 | |

- **暖跑 predict 命中率 14/14 = 100%**（先验给的预期照抄即中——验证信号学来的知识是准的）。
- T1 token 微涨：先验注入本身 ~500 token，任务太小时吃掉节省；**任务越长先验摊销越划算**（T2 已见 -23%）。
- 全部 4 组配置 × 12 运行 = **48 次运行 100% completed、写全 verified**——顺带的鲁棒性数据。

## 追加：T3「产生对象的动作」页型（先验可迁移性修复的验证，reps=2）

定稿配置 D 之后自查发现：先验注入的是 diff **原文**（含实例 id，如
`object appeared: object:apply:1`），模型照抄当 predict 后跨实例必落空——
"动作产生对象"这一最常见页型上，先验会反向做功。三修（泛化注入 + 注入上限 8 行
active 优先 + `matchesPrediction` 对象实例宽容 + 换页补注）后的 T3 数字：

| 指标 | 冷 | 暖 | Δ |
|---|---|---|---|
| LLM 往返 | 6.50 | 3.50 | **-46%** |
| token（p+c） | 11708 | 6793 | **-44%** |
| predict 命中 | 2/3（修匹配前 0/3） | **3/3（跨实例）** | |
| completed | 2/2 | 2/2 | |

- 修匹配前冷跑 predict 0/3 全落空——模型从 toolResult 证据照抄带 id 的预测，
  下一个实例出现即假 mispredict（白白中断投机）；实例宽容后 2/3。
- `live-drift.ts` 回归通过：实例宽容只到 id 层，形状级行为漂移照常落空/上报/自愈。

## 过程中抓到的真 bug（bench 的第二产出）

配置 B 首跑即触发 **OpenAI 协议 400**：批次中断（落空/未验证/错误/取消）跳过的 tool_calls 没有 tool 回执，下一轮请求被拒。切片8 时模型极少批量所以从未暴露——"全绿但崩"再添一例。已修（跳过步骤补 SKIPPED 回执）+ 确定性测试守卫（`loopLookahead.test.ts`）。修后配置 C/D 的冷跑多次真实 mispredict 中断，循环全部正常恢复——中断恢复路径在 live 下过了。

## 诚实边界

- N=3/格、单模型（deepseek-v4-pro）、小页面短任务——数字是方向性的，不是统计显著的。
- 配方先验（程序模式 RecipeBook）的 A/B **仍未量化**，此报告只覆盖读循环的世界模型先验。
- 冷暖对比不含跨会话噪音（同进程内存先验）；跨会话（fromJSON 读盘）路径未在本 bench 内计量。
