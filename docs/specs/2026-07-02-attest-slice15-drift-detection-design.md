# 切片15：漂移检测——把验证信号吃第四口

> 2026-07-02。用户定调：先补洞（切片14 写路径加固），再上一个**提高核心价值**的新机制。
> 核心价值 = 廉价确定性 diff 验证信号一鱼三吃（安全闸 / 投机许可证 / 学习信号）。
> 本切片让同一信号长出第四个用途：**漂移检测器**——同时兑现 Attest 对 VOIX
> 定位承诺的第三根柱子（outcome 验证 / 信任 / **漂移**）中此前只有被动防线的那根。

## 机制

同一签名下，已知动作不再产生已知效果 = 页面行为漂移的**确定性证据**。全部数据来自
已有账本流（verified diff），零新增采集成本。

- **写时裁定**：每次真正执行的写（含"执行了但无变化"）落账时，内核立刻裁定
  世界模型旧条目 命中/落空/漂移。绝不把陈旧条目丢给模型自己判断。
- **两级阈值**：第 1 次落空 → `suspect`（先验仍注入但带警示"最近一次未按此效果
  发生，先验证再依赖"）；连续第 2 次落空 → 判**漂移**：上报 `DriftEvent`（新增
  AgentStep `drift`，repl 可视）+ 写时自愈——有新行为就采纳为新先验、无行为就逐出。
  命中即回 `active` 清零。
- **形状比较（精确优先）**：漂移判定不用原始 diff 串——`object appeared:
  object:task:9` 与 `…task:10` 是同构（同动作不同参数的实例差异），比较剥掉实例
  id/具体值的**结构形状**；存储仍保原文给模型看。命中 = 已知形状 ⊆ 实际形状
  （页面多做别的不算落空，与 `matchesPrediction` 的满足档语义一致）。
- **负样本学习**：无先验的动作被确认"执行了但无可观察效果"≥2 次 → 注入反先验
  "已知多次执行均无可观察变化，勿依赖它达成效果"。先验从"只说哪条路通"升级为
  "也说哪条路死"。
- **持久化**：`toJSON` 升 v2（含 misses/noEffect），`fromJSON` 兼容 v1 旧盘格式。

## 研究谱系（2026-07 检索，各取一件东西）

- **DDM/Page-Hinkley/ADWIN**（数据流概念漂移经典）：基于错误率的双阈值
  warning/drift 模式 → 我们的 suspect/漂移两级。经典方法要 ground-truth 标签，
  我们的 diff 验证**天生就是标签**——这正是它们能干净落到这里的原因。
- **STALE（arXiv 2605.06527）**：记忆失效的"状态裁定缺口"——新旧记忆共存而无
  仲裁是高失败率主因；写时裁定远胜查询时临时协调；"授权输出"只注入活跃记忆。
  另一发现"认知与行动脱节"（模型识别出陈旧≠会按新状态行动）支持裁定必须由
  内核确定性完成，不委托模型。
- **SkillGuard（arXiv 2605.10990）**：drift = 角色依赖的契约违反；精确优先
  （0 FP 哲学）+ 保守分层匹配 → 我们只用确定性证据（verified diff 形状不符 /
  确认无变化）判漂移，实例差异绝不误报。
- **Library Drift（arXiv 2605.19576）**：自进化技能库的陈旧注入 + 检索稀释会把
  agent 拖到**低于无库基线**——漂移条目必须主动逐出/自愈，不能只标记留存。
- **WMA / WebDreamer（arXiv 2410.13232 等）**：世界模型 lookahead 一系用 LLM
  模拟状态转移（会幻觉）；Attest 的转移知识从账本证据学，**grounded、不模拟**——
  这是差异化，形状抽象与 WMA 的 transition-focused observation abstraction 同构。

## 红线自查

- 记忆只加速不背书：先验（含 suspect 警示/负先验）只注入上下文；verifier 仍是
  唯一真相；漂移自愈只改先验、绝不改判定。
- 判定链不动：verify-or-refuse / held / computeOutcome 均未触碰。
- LLM 仍主导：漂移裁定是内核对**自己记忆**的管理，不替模型做任何页面决策。
